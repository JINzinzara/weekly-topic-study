<a name="topic-1"></a>
## Topic 1. 리사이징(Resizing)과 정규화(Normalization)를 하는 이유

### 1. 내용

#### 리사이징 (Resizing)
- **정의:** 이미지의 가로/세로 크기를 원하는 크기로 조정하는 전처리 과정
- **이유:** 1. 대부분의 딥러닝 모델은 고정된 크기의 입력 텐서를 요구한다
           2. 입력의 형식을 통일함으로써 안정적인 학습이 가능하다
- **주의할 점:** 1. 너무 큰 해상도: 연산 비용 및 메모리 사용량 증가
               2. 너무 작은 해상도: 정보 손실

#### 정규화 (Normalization)
- **정의:** 데이터의 스케일을 조정하는 과정
- **이유:**
  - **머신러닝**
    1. 모델의 안정성 향상
    2. 학습 속도 증가
    3. 특성 간 단위 차이 해소
   
  - **딥러닝**
    1. 내부 공변량 변화 감소
    2. 그래디언트 소실/폭발 문제 완화
    3. 일반화 성능 향상
    4. 학습의 안정성 향상

- **주요 방법:** 

<**머신러닝**>
| 정규화 방법 | 특징 |
|-----------|------|
| Min-Max 정규화 | 데이터를 [0, 1] 범위로 스케일링 |
| tandardization | 평균을 0, 표준편차를 1로 조정 |

<**딥러닝**>
| 정규화 방법 | 특징 |
|-----------|------|
| Batch Normalization | 신경망의 각 층에서 입력을 정규화 |
| Layer Normalization | 각 샘플에 대해 모든 뉴런의 출력을 정규화 |
| Instance Normalization | 스타일 전이 작업에서 사용하는 정규화 |
| Group Normalization | 채널을 여러 그룹으로 나누고, 각 그룹에서 정규화(BN의 한계 극복) |
| RMS Normalization | 각 샘플에 대해 RMS 계산하여 정규화(언어 모델에 사용) |

#### 머신러닝 vs 딥러닝 정규화 비교
| 구분 | 머신러닝 | 딥러닝 |
|------|--------|--------|
| 적용 시점 | 전처리 단계(데이터셋 전체) | 각 레이어 |
| 적용 범위 | 입력 특성 | 중간 레이어의 활성화 값 |
| 학습 가능성 | 고정된 변환 | 학습 가능한 파라미터 포함 |
| 목적 | 데이터 스케일 조정 | 그래디언트 소실/폭발 문제 해결, 내부 공변량 변화 감소 등 |

## Topic 2. 데이터 증강(Data Augmentation)이란?

### 1. 내용

#### 데이터 증강이란?
- **정의:** 데이터에 약간의 변형을 가해 새로운 학습 샘플을 생성
- **목적:** 과적합을 줄이고 모델의 일반화 성능 향상

#### 데이터별 주요 증강 기법
- **이미지 증강**
| 기하학적 변환 | 광도 변화 | 이미지 혼합 |
|----------|------|------|
| 좌우 반전 (Horizontal Flip) | 밝기 조절 (Brightness) | Cutout |
| 상하 반전 (Vertical Flip) | 색상 변환 (Color Jitter) | Mixup |
| 회전 (Rotation) | | CutMix|
| 크롭 (Random Crop) | | |
| 노이즈 추가 (Noise) | | |

- **텍스트 증강**
| 규칙 기반 | 신경망 방법 | 혼합 텍스트 |
|---------|----------|----------|
| 무작위 삭제/삽입 | EDA (Easy Data Augmentation) | Mixup |
| 동의어 교체 | BERT 마스킹 | SentMix |
| 문장 반전/수동화 | GPT 생성 | CutMix |
| 역번역 | 조건부 생성 (Conditional Generation) | |

#### 데이터 증강 시 주의사항
- 과도한 데이터 증강은 오히려 학습을 방해
- 레이블 일관성 유지
- 데이터 도메인에 맞는 증강 사용
- 테스트셋에서는 TTA가 아닌 이상 증강 사용 금지

## Topic 3. Transfer Learning(전이 학습)이란?

### 1. 내용

#### 전이 학습이란?
- **정의:** 대규모 데이터셋으로 학습된 모델의 지식을 사용하여 특정 문제에 전이하여 활용하는 것
- **핵심 아이디어:** 이미 학습된 특징 표현을 재사용함으로써, 처음부터 학습하지 않아도 좋은 성능을 낼 수 있다
- **언제 사용하나:**
  1. 타겍 도메인의 데이터가 부족할 떄
  2. 학습 시간/자원을 절약하고자 할 때
  3. 소스 도메인과 타겟 도메인이 유사한 특징을 공유할 떄

#### 전이 학습의 유형

| 방법 | 설명 | 예시 |
|------|------|----------|
| 유도 전이 | 대상 도메인과 소스 도메인 간의 차이점/유사성에 관계없이 소스 작업과 대상 작업이 다른 경우 | 멀티테스크 학습 |
| 비지도 학습 | 유도 전이와 비슷하지만 소스/대상 데이터에 라벨이 지정되지 않은 경우 | 이상 탐지 |
| 변환적 전이 | 소스 작업과 대상 작업이 동일하지만 도메인이 다른 경우 | 식당 후기 분류 학습 &rarr; 영화 후기 분류 적용 |

#### 대표적인 사전 학습 모델
| 이미지 | 자연어 |
|------|------|
| VGG16 / VGG19 | BERT |
| ResNet | GPT 시리즈 |
| EfficientNet | RoBERTa |
| MobileNet | T5 |
| Vit | KoELECTRA(한국어 특화) |

#### 전이 학습 vs 처음부터 학습 비교
| 구분 | 전이 학습 | 처음부터 학습 |
|------|--------|------------|
| 필요 데이터 양 | 적어도 가능 | 많음 |
| 학습 시간 | 짧음 | 긺 |
| 성능 | 데이터가 부족하거나, 도메인이 유사할 경우 효과적 | 데이터가 충분하고 도메인 특화적 |
| 적합한 상황 | 데이터 부족, 유사 도메인 | 대규모 데이터 보유, 새로운 도메인 |

### 3. 왜 중요한가?
- 기존 모델의 지식을 재활용하여 적은 비용으로 높은 성능을 달성 가능
