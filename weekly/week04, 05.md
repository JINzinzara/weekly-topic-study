## Topic 1. 딥러닝과 머신러닝의 포함관계

### 1. 핵심 개념 요약
> 머신 러닝과 딥 러닝은 데이터를 학습하고 예측하는 기술이지만, 머신 러닝은 더 간단한 문제에 적합하고, 딥 러닝은 복잡한 데이터 분석에 뛰어난 성능을 발휘한다

### 2. 상세 내용

#### 포함 관계
- **AI ⊃ 머신러닝 ⊃ 딥러닝**

#### 머신러닝 (Machine Learning)
- **정의:** 컴퓨터가 데이터를 학습하여 스스로 특정 작업을 수행할 수 있도록 하는 기술
- **특징:** 컴퓨터 스스로가 데이터의 패턴을 학습하고, 학습을 바탕으로 결정 또는 예측
          - 데이터 기반 학습
          - 알고리즘의 다양성
          - 훈련 방법: 비지도, 지도, 강화 학습
- **예시:** Desicion Tree, K-NN, SVM

#### 딥러닝 (Deep Learning)
- **정의:** 인간의 뇌 구조를 바탕으로 만들어진 인공신경망을 이용하여 컴퓨터가 더 복잡한 학습을 할 수 있도록 하는 기술
- **특징:** 이미지, 음성, 자연어 처리와 같은 복잡한 데이터 분석에 뛰어난 성능
          - 다층 신경망(MLP)
          - 자동 특징 추출
          - 대량의 데이터
- **예시:** ANN, CNN, RNN, Transformer

#### 머신러닝 vs 딥러닝 비교
| 구분 | 머신러닝 | 딥러닝 |
|------|--------|--------|
| 특징 추출 | 사람이 직접 | 자동으로 학습 |
| 데이터 양 | 상대적으로 적은 데이터 | 대량의 데이터 |
| 계산 비용 | 상대적으로 낮음 | GPU 필요함 |
| 문제 해결 | 정형화된 데이터 | 비정형화된 데이터 |
| 활용 예시 | 스팸 메일 분류, 추천 시스템 | 얼굴 인식, 음성 인식, 자율 주행 |

<a name="topic-2"></a>
## Topic 2. 딥러닝 하이퍼파라미터의 종류

### 1. 핵심 개념 요약
하이퍼파라미터는 사람이 직접 정해주는 값으로 적절한 하이퍼파라미터 설정을 통해 모델의 성능을 크게 향상시킬 수 있다

### 2. 상세 내용

#### 하이퍼파라미터란?
- **정의:** 학습 전 사람이 지정해주는 매개변수(parameter)이다
- **파라미터와의 차이:** 

#### 파라미터 vs 하이퍼파라미터
| 구분 | 파라미터 | 하이퍼파라미터 |
|------|--------|-------------|
| 결정 방식 | 학습을 통해 자동 결정 | 사람이 직접 설정 |
| 예시 | 가중치(weight), 편향(bias) | 학습률(learning rate), epoch, 배치 크기(batch size), 활성화 함수 |

#### 주요 하이퍼파라미터 종류

| 하이퍼파라미터 | 설명 | 영향 |
|-------------|------|------|
| 학습률 (Learning Rate) | 다음 경사 이동할 때의 스텝 크기 | 크면 발산, 적으면 학습 시간 증가 |
| 배치 크기 (Batch Size) | 가중치를 한 번 업데이트할 때 사용하는 데이터의 개수 | 학습 시간 |
| 에폭 (Epoch) | 모든 데이터를 학습 완료한 상태 | 크면 과적합, 적으면 충분한 학습 X |
| 옵티마이저 (Optimizer) | 학습을 실제로 수행하는 프로그램 | 안장점 문제와 방향 변동성 해결 |
| 드롭아웃 비율 (Dropout Rate) | 학습 시 일부 뉴런을 무작위로 출력 0으로 만듦 | 과적합 방지 |
| 활성화 함수 (Activation Function) | 뉴런의 가중합을 받아 최종 출력으로 반환하는 함수 | 비선형성을 추가하여 더 복잡한 패턴 학습 가능 |

### 3. 왜 중요한가? 
- 학습률, 배치 크기, 에폭 크기 등은 학습의 안정성과 속도를 결정
- 활성화 함수, 손실 함수, 드롭아웃 비율은 학습 능력과 성능을 결정

### 1. 내용

#### PyTorch
**-특징:** 코드가 길고 복잡하지만, 세세한 부분을 수정하거나 오류 원인 찾기 쉬움

## Topic 4. PyTorch 텐서 vs NumPy 배열

### 1. 핵심 개념 요약
텐서와 numpy.array는 다차원 배열 데이터 구조를 같는 것에서 동일하지만, GPU 연산과 자동 미분 계산 여부에 의해 나눠진다

### 2. 상세 내용

#### 텐서 (Tensor)란?
- **정의:** 다차원 배열을 표현하는 데이터 구조
- **특징:** 딥러닝과 같은 고성능 계산에 최적화된 기능을 제공

#### NumPy 배열 (Array)이란?
- **정의:** 다차원 배열을 표한하는 데이터 구조
- **특징:** CPU를 활용한 일반 수학 계산에 최적화된 기능을 제공

#### 텐서 vs NumPy 배열 비교
| 구분 | PyTorch 텐서 | NumPy 배열 |
|------|------------|----------|
| GPU 연산 | 지원 | 미지원 |
| 자동 미분 | 지원 (autograd) | 미지원 |
| 용도 | 일반 수학 및 데이터 분석 | 고성능 연산 작업 |


#### 텐서 ↔ NumPy 변환 방법
```python
import torch
import numpy as np

# NumPy → 텐서
arr = np.array([1, 2, 3])
tensor = torch.from_numpy(arr)

# 텐서 → NumPy
tensor = torch.tensor([1, 2, 3])
arr = tensor.numpy()
```

### 3. 왜 중요한가? 
- GPU 연산과 자동 미분 기능을 통해 텐서가 딥러닝 모델 개발의 표준이 되었다
