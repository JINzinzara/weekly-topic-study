## Topic 1. 지도 학습과 비지도 학습의 차이

### 1. 핵심 개념 요약
머신러닝 접근 방식에는 지도 학습과 비지도 학습, 강화 학습이 있다.
그 중 지도 학습은 레이블이 주어진 상태에서 학습이고,
비지도 학습은 레이블이 주어지지 않은 상태에서 스스로 학습한다.


### 2. 상세 내용

#### 지도 학습 (Supervised Learning)
- **정의:** 데이터 분석을 할 때 레이블(정답)을 알려주고 학습시키는 머신러닝 접근 방식
- **특징:** 데이터를 분류하거나 결과를 정확하게 예측하도록 설계
- **대표 알고리즘:**

| 종류 | 작동 방식 |
|-----|---------|
| 회귀 | 종속변수와 독립변수 간의 관계를 이해 |
| 분류 | 데이터를 특정 카테고리에 할당 |

#### 비지도 학습 (Unsupervised Learning)
- **정의:** : 데이터 분석 시, 레이블을 알려주지 않고 무수히 많은 데이터를 통해 스스로 학습시키는 머신러닝 접근 방식
- **특징:** : 스스로 데이터 간의 숨겨진 패턴을 파악
- **대표 알고리즘:**
 
| 종류 | 작동 방식 |
|-----|---------|
| 클러스터링 | 데이터의 유사성 또는 차이점에 따라 그룹화 |
| 연관 | 변수간의 관계 파악 |
| 차원 축소 | 원시 데이터를 유지하며 데이터 입력 수를 관리 가능한 크기로 축소 |

#### 비교 정리
| 구분 | 지도 학습 | 비지도 학습 |
|------|----------|------------|
| 레이블(정답) | 필요 | 불필요 |
| 목적 | 새로운 데이터의 결과 예측 | 많은 양의 데이터에 대한 통찰 |
| 복잡성 | 간단함 | 복잡함 |
| 활용 예시 | 스팸탐지, 일기예보 | 이상 감지, 추천 엔진|

**현장에서는 지도/비지도를 하이브리드로 사용함**

## Topic 2. 손실 함수(Loss Function)란?

### 1. 핵심 개념 요약
모델의 틀린 정도를 나타내는 함수(모델 성공 측정)
변수를 최적화하여 손실을 최고화하는 알고리즘을 모델에 입력

### 2. 상세 내용

#### 정의
- 손실 함수란 예측한 값과 실제 값 사이의 차이를 측정하는 함수
- 손실 함수의 값을 최고화 하는 것이 모델 학습의 목표
- 레이블이 정해진 *지도 학습*에서 사용

#### 왜 필요한가?
- 모델의 성능을 평가하고, 앞으로의 개선 방향성을 알려줌

#### 주요 손실 함수 종류
| 손실 함수 | 수식 | 언제 사용 |
|----------|------|----------|
| MSE (Mean Squared Error) | np.mean((y - y_pred)**2) | 회귀 |
| RMSE (Root Mean Squared Error) | np.sqrt(mse) | 회귀 |
| MAE (Mean Absolute Error) | np.mean(|y - y_pred |) | 회귀 |
| R² (결정 계수) | 1 - (SSE / SST) | 회귀 |
| Binary Cross-Entropy | -np.mean(y * np.log(y_pre + eps) + (1 - y) * np.log(1 - y_pred + eps)) | 이진 분류 |

#### 손실 함수와 최적화의 관계
- 최적화는 모델이 실제 값과 예측 값 사이의 오차를 줄여가기 위해 가중치를 조정하는 과정으로, 경사하강법을 사용하여 모델의 성능을 향상 시킨다
- 과적합(Overfitting): 모델이 학습 데이터를 과도하게 잘 맞추려는 경향으로, 데이터를 전부다 외워버리는 상태
  문제점: 새로운 데이터에 대한 일반화 성능이 떨어짐
- 과소적합(Underfitting): 모델이 학습 데이터의 패턴을 충분히 학습하지 못한 상태
  문제점: 학습 데이터에 대한 일반화는 물론, 테스트 데이터에 대한 예측 성능도 떨어짐

  
### 3. 왜 중요한가?
- 모델을 학습시켜 정확한 예측 결과를 산출하는 머신 러닝의 기본 목표 달성 가능

## Topic 3. 편향(Bias)과 분산(Variance)

### 1. 핵심 개념 요약
모델의 오류의 원인으로 편향과 분산 두 가지가 있다
고편향 모델을 과소적합을 초래하며, 고분산 모델은 과적합을 초래한다


### 2. 상세 내용

#### 편향 (Bias)
- **정의:** 단순한 모델로 인한 오류
- **특징:** 데이터를 하나의 직선에 맞추려고하여, 모델을 단순화시키는 오류를 발생
- **높을 때 나타나는 현상:** 과소적합

#### 분산 (Variance)
- **정의:** 복잡한 모델로 인한 오류
- **특징:** 데이터의 패턴뿐만아니라 노이즈까지 학습하려하는 오류를 발생
- **높을 때 나타나는 현상:** 과적합

#### 편향-분산 트레이드오프 (Bias-Variance Tradeoff)
- 편향과 분산은 상충관계(서로 반대 방향)이다
- 적절한 균형을 잡기 위한 **정규화**


### 3. 왜 중요한가? 
- 모델의 예측 오류를 결정하는 핵심 요소

## Topic 4. K-폴드 교차 검증에서 K 선택 기준

### 1. 핵심 개념 요약
K-폴드 교차 검증은 모델의 과적합을 방지하기 위해 일반화 성능을 측정하는 통계적 평가 방법으로,
데이터의 수와 계산 비용에 따라 K 값을 선택할 수 있다. 가장 보편적인 K 값은 5~10이다.



### 2. 상세 내용

#### 교차 검증(Cross Validation)이란?
- 모델의 일반화 성능을 측정하기 위한 통계적 평가 방법

#### K-폴드 교차 검증 작동 방식
- K개의 데이터 폴드를 생성 &larr; K번 만큼 각 폴드 세트에 학습/검증 평가 반복

#### K 값 선택 시 고려사항
| 고려사항 | 설명 |
|----------|------|
| 데이터 크기 | 데이터 크기에 따라 적절한 K 값 |
| 계산 비용(시간/자원) | 주어진 예산에 따라 최적의 K 값 선정 |
| 편향-분산 트레이드오프 | 小K 값&larr; 과소적합, 大K 값&larr; 과적합 |

#### 일반적인 K 값 선택 기준
- **K=5:** 가장 많이 사용되는 값으로 계산 비용/평가 신뢰성 균형 좋음
- **K=10:** 데이터의 수가 충분할 경우, 더 높은 신뢰성 필요시 사용
- **K=데이터 수 (LOOCV):** 데이터의 수가 적을 때 사용. 신뢰성은 높으나, 계산 비용이 큼

### 3. 왜 중요한가?
- 모델의 과적합을 해결
